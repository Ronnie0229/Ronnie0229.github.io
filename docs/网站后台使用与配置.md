# 网站后台使用与配置

## 后台地址

- 管理首页：`https://ronniecross.com/admin/`
- 文章管理：`https://ronniecross.com/admin/editor.html`
- 留言管理：`https://ronniecross.com/admin/comments.html`

后台必须先经过 Cloudflare Access 验证，只有管理员邮箱可以进入。

## 第一次启用

### 1. 创建 GitHub OAuth App

进入 GitHub：

1. 点击头像。
2. 打开 `Settings`。
3. 打开 `Developer settings`。
4. 打开 `OAuth Apps`。
5. 点击 `New OAuth App`。

填写：

- Application name：`RonnieCross Admin`
- Homepage URL：`https://ronniecross.com`
- Authorization callback URL：`https://ronniecross.com/admin/api/callback`

创建后记录 `Client ID`，并生成一个 `Client secret`。不要把 secret 写入代码或聊天记录。

### 2. 配置 Cloudflare 环境变量

进入 Cloudflare Pages 项目 `ronniecross` 的设置，在生产环境添加：

- `GITHUB_CLIENT_ID`：GitHub OAuth App 的 Client ID
- `GITHUB_CLIENT_SECRET`：GitHub OAuth App 的 Client secret，使用加密变量
- `ADMIN_EMAIL`：允许进入后台的邮箱
- `CF_ACCESS_TEAM_DOMAIN`：Cloudflare Zero Trust 团队域名，例如 `https://example.cloudflareaccess.com`
- `CF_ACCESS_AUD`：后台 Access 应用的 Application Audience (AUD)

修改环境变量后重新部署一次。

### 3. 配置 Cloudflare Access

在 Cloudflare Zero Trust 中创建一个 Self-hosted 应用：

- 域名：`ronniecross.com`
- 路径：`/admin/*`

策略只允许 `ADMIN_EMAIL` 中填写的邮箱。

成功标志：打开 `/admin/` 时会先出现 Cloudflare 登录验证，其他邮箱无法进入。

## 日常使用

### 在手机上发布文章

1. 打开 `/admin/`。
2. 进入“文章管理”。
3. 使用 GitHub 登录。
4. 新建文章。
5. 填写标题、摘要、日期、分类、经文和正文。
6. 内容没完成时保持“保存为草稿”开启。
7. 准备发布时关闭“保存为草稿”，然后保存。
8. 等待 Cloudflare Pages 自动部署，一般需要几分钟。

文章标题可以修改。旧文章不要手工修改文件名，原网址、留言和阅读次数就会继续保留。

### 管理留言

1. 从 `/admin/` 进入“留言管理”。
2. 可以按昵称、内容、文章地址和状态查询。
3. “隐藏留言”会同时隐藏它下面的回复，可以恢复。
4. “永久删除”会同时删除它下面的回复，不能恢复。

优先使用隐藏，只有确定不再需要时才永久删除。

## 在家用电脑编辑

每次开始本地编辑前，运行：

```shell
npm run sync
```

脚本只有在本地没有未提交修改时才会执行 `git pull --ff-only`。如果本地正在编辑，它会停止，不会覆盖文件。

完成本地文章修改后，正常提交并推送到 GitHub。不要让电脑和手机同时修改同一篇文章。

## 数据位置

- 文章和图片：GitHub 仓库
- 留言、回复和阅读次数：Cloudflare D1
- 后台登录：Cloudflare Access
- 文章编辑授权：GitHub OAuth

文章和留言虽然分别存放，但在 `/admin/` 中统一管理。
