# 项目目录说明

## 目录用途

- `docs/`：长期说明、方案、运行手册和内容整理报告。
- `src/`：Astro 正式代码与已发布的 Markdown 文章。
- `scripts/`：采集、清洗、导入、维护脚本和数据库迁移文件。
- `data/raw/`：原始资料，只新增，不直接改写。
- `data/processed/`：从原始资料整理出的中间结果。
- `data/export/`：需要单独交付的最终导出成果。网站构建仍按 Astro 约定输出到根目录 `dist/`。
- `assets/`：网站图片、模板和静态素材，由 Astro 直接作为静态资源提供。
- `logs/`：运行日志，不提交到 Git。
- `tmp/`：可随时清理的临时文件，不提交到 Git。

## 必须保留在根目录的内容

以下内容受 Astro、Cloudflare Pages、npm 或 GitHub 约定限制，不移动到分类目录：

- `package.json`、`package-lock.json`
- `astro.config.mjs`、`tsconfig.json`、`wrangler.jsonc`
- `functions/`：Cloudflare Pages Functions 固定入口。
- `.github/`、`.git/`、`.gitignore`
- `node_modules/`、`.astro/`、`dist/`：依赖、缓存和构建输出。

## 资料处理流程

1. 新资料放入 `data/raw/教会讲道/` 或 `data/raw/分享/`。
2. 运行 `scripts/` 中对应的导入脚本。
3. 中间结果写入 `data/processed/`。
4. 可发布文章同步到 `src/content/posts/`。
5. 整理报告写入 `docs/内容整理报告/`。
