# AGENTS.md

## 项目定位

- 本仓库是个人文章网站的唯一正式代码仓库。
- 网站开发、分享资料整理、讲道资料整理和发布都在本仓库完成。
- 日常开发应使用本机 SSD 上的 Git 工作副本，不要直接在 NAS 目录运行 Git、Node、Codex 或文件监听。

## 内容入口

- 分享收件目录：`\\RonnieNAS\tmp\分享`
- 讲道收件目录：`\\RonnieNAS\tmp\讲道`
- 分享原始资料：`data/raw/分享/`
- 讲道原始资料：`data/raw/教会讲道/`
- 分享中间稿：`data/processed/整理后的分享文章/`
- 讲道中间稿：`data/processed/整理后的讲道文章/`
- 正式网站文章：`src/content/posts/`

## NAS 安全规则

1. 首次访问 NAS 路径时先做只读目录检查。
2. 普通文件操作直接使用 UNC 或 SMB 路径，不重复挂载盘符。
3. `\\RonnieNAS\share` 是受保护归档区：可以新增和读取，不移动、不删除、不覆盖已有资料。
4. 新资料移入 `data/raw/` 后必须用 SHA-256 验证源文件与目标文件一致，再删除收件目录中的源文件。
5. 讲道中文整理稿复制到 `share` 后，必须再次验证 SHA-256、文件数和字节数。

## 发布要求

- 用户说明“分享”时走分享流程；说明“讲道”时走讲道流程。
- 讲道资料必须确认日期、经文、标题和讲员；无法可靠判断时先询问。
- 每次内容任务默认完成整个流程：归档、整理、生成文章、更新报告、构建、提交、推送和线上验证。
- 导入脚本不得删除或覆盖另一类别的文章。
- `node_modules/`、`.astro/`、`dist/` 和 Python 虚拟环境只保留在本机，不同步到 NAS 或 Git。
